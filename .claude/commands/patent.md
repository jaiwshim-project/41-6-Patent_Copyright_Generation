# 특허출원 AI 프로세스 스킬

특허 출원 전 과정(기술 분석 → 청구항 설계 → 명세서 작성 → 도면 생성)을 80개 명령어로 체계적으로 진행합니다.

## 🚀 실행 방법

```
/patent              — 전체 진행 상황 + 다음 단계 추천
/patent p1           — Process 1 (기술 분석, 명령 1~9)
/patent p2           — Process 2 (경쟁기술 분석, 명령 10~11)
/patent p3           — Process 3 (청구항 설계, 명령 12~29)
/patent p4           — Process 4 (명세서 작성, 명령 30~41)
/patent p5           — Process 5 (명세서 완성, 명령 45~52)
/patent p6           — Process 6 (실시예/검증, 명령 53~62)
/patent p7           — Process 7 (최종 검수, 명령 42~44, 63~65)
/patent p8           — Process 8 (도면 설계/생성, 명령 66~80)
/patent cmd 20       — 특정 명령어 (예: 명령 20 = V3 아키텍처 생성)
```

## 📁 작업 폴더

현재 디렉토리에 `patent-output/` 폴더를 자동 생성하여 다음과 같이 구성합니다:

```
patent-output/
├── p1-tech/          (명령 1~9 결과물)
├── p2-competitor/    (명령 10~11 결과물)
├── p3-claims/        (명령 12~29 결과물)
├── p4-spec-draft/    (명령 30~41 결과물)
├── p5-spec-complete/ (명령 45~52 결과물)
├── p6-embodiment/    (명령 53~62 결과물)
├── p7-final/         (명령 42~44, 63~65 결과물)
└── p8-drawings/      (명령 66~80 결과물)
```

## ✅ 실행 가이드

**처음 시작할 때:**
1. 작업 폴더 경로 확인 (또는 현재 디렉토리)
2. `/patent p1` 또는 `/patent` 입력
3. 각 명령어를 순서대로 실행 (명령 1→명령 2→...)
4. 파일 생성이 필요한 명령어는 자동으로 `patent-output/` 폴더에 저장됨

**중간에 다시 시작할 때:**
1. `/patent` 입력
2. 이전에 저장된 파일들이 자동으로 감지됨
3. 완료되지 않은 다음 명령어부터 진행

**특정 프로세스 또는 명령어만 실행할 때:**
- `/patent p3` — Process 3 (청구항 설계)부터 시작
- `/patent cmd 20` — 명령 20만 실행

## 📊 파일 저장 규칙

- **MD 파일**: 텍스트 결과물 (분석, 평가, 요약, 구조 등)
- **SVG 파일**: 시각화 결과물 (아키텍처, 도면 등)
- **자동 저장**: 명령어 실행 후 결과물은 즉시 `patent-output/` 에 저장됨
- **폴더 구조**: 프로세스별로 자동 생성 및 정렬
