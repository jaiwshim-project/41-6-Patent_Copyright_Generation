# 특허 명세서 작성 자료 생성 스킬

프로젝트의 소프트웨어 발명(알고리즘, 방법론, 시스템 구조)에 대한 **특허 출원용 명세서 초안 및 도면**을 자동으로 생성한다.
한국특허청(KIPO) 전자출원 시스템(특허로)에 제출 가능한 형식으로 출력한다.

> **주의**: 이 스킬은 명세서 **초안**을 생성합니다. 최종 출원 전 변리사 검토를 강력히 권장합니다.

---

## 🎯 개요

이 스킬은 다음 작업을 자동으로 수행합니다:

1. **프로젝트 분석** — 소스코드 구조, Git 이력 분석
2. **발명 식별** — 특허 출원 가능한 발명 요소 자동 추출
3. **명세서 생성** — 12개 초안 문서 + 도면 자동 작성
4. **청구범위 작성** — 독립항 + 종속항 자동 초안
5. **선행기술 조사** — Google Patents 검색 자동화

---

## 📋 Phase 1: 발명 분석

### Step 1: 프로젝트 스캔 및 발명 후보 식별

현재 작업 디렉토리를 스캔하여 특허 출원 가능한 발명 요소를 자동 식별한다.

**발명 후보 탐색 대상:**
- 고유 알고리즘 (정렬, 검색, 추천, 최적화 로직)
- 데이터 처리 방법 (파이프라인, 변환, 집계 로직)
- 시스템 아키텍처 (모듈 구성, 통신 방식, 계층 구조)
- 방법론/프레임워크 (개발 프로세스, 워크플로우)
- UI/UX 인터랙션 방법 (독창적 사용자 인터페이스 동작)
- AI/ML 모델 활용 방법 (프롬프트 엔지니어링, RAG 파이프라인)
- 하드웨어-소프트웨어 연동 방법 (IoT, 센서 처리)

### Step 2: 발명 유형 판정

| 유형 | 설명 | 출원 카테고리 | 예시 |
|------|------|-------------|------|
| **방법 발명** | 시간 순서에 따른 처리 단계 | 방법 청구항 | "~하는 방법" |
| **시스템 발명** | 구성요소와 연결 관계 | 장치/시스템 청구항 | "~하는 시스템" |
| **컴퓨터 판독 가능 기록매체** | 프로그램이 기록된 매체 | 매체 청구항 | "~프로그램이 기록된 매체" |
| **복합 발명** | 방법 + 시스템 동시 | 독립항 2개 이상 | 가장 일반적 |

### Step 3: 발명 정보 수집

사용자에게 다음 정보를 확인한다:

```
📋 특허 출원 정보 입력
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. 발명자명 (본명)
2. 발명자 영문명
3. 발명자 주소
4. 출원인 (본인/법인)
5. 발명의 명칭 (한글)
6. 발명의 명칭 (영문)
7. 기술 분야 (예: 인공지능, 웹 서비스)
8. 해결하려는 과제 (한 문장)
9. 공동 발명자 유무
10. 선행기술 조사 여부
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 📄 Phase 2: 명세서 자료 생성

### 생성 파일 목록

```
{프로젝트루트}/patent/
├── 01_발명의_명칭.md               ← 한글/영문 발명 명칭
├── 02_기술분야.md                  ← IPC 분류 및 기술 분야
├── 03_배경기술.md                  ← 종래 기술의 문제점
├── 04_해결하려는_과제.md            ← 본 발명의 목적
├── 05_과제의_해결수단.md            ← 핵심 기술 구성 (청구항 기초)
├── 06_발명의_효과.md               ← 정량적/정성적 효과
├── 07_발명의_상세한_설명.md         ← 실시예 포함 상세 설명
├── 08_청구범위_초안.md              ← 독립항 + 종속항
├── 09_요약서.md                    ← 400자 이내 요약
├── 10_도면/                        ← 도면 파일들
│   ├── 도면1_시스템구성도.svg
│   ├── 도면2_처리흐름도.svg
│   ├── 도면3_데이터흐름도.svg
│   └── 도면_설명.md
├── 11_선행기술_조사.md              ← 유사 특허 검색 결과
└── 12_체크리스트.md                 ← 출원 전 확인 사항
```

### 파일별 생성 규칙

#### 01. 발명의 명칭
- 발명의 대상(방법/시스템/장치)을 명시
- 핵심 기술 키워드를 포함, 간결하되 기술 범위를 특정
- 예: "지식 그래프 기반 검색 증강 생성을 이용한 소셜 미디어 의견 시뮬레이션 시스템 및 방법"

#### 02. 기술분야
- IPC(국제특허분류) 코드 자동 추정
- AI/ML → G06N 20/00
- 웹 서비스 → G06F 16/958
- 데이터베이스 → G06F 16/00
- 자연어 처리 → G06F 40/00
- 전자상거래 → G06Q 30/00

#### 03. 배경기술
- README/CLAUDE.md에서 프로젝트 목적/배경 추출
- 종래 기술 설명 → 종래 기술의 문제점 3가지 도출
- 선행기술 문헌 (특허문헌 + 비특허문헌) 목록

#### 04. 해결하려는 과제
- 배경기술 문제점 1:1 대응하는 발명의 목적 기술
- 정형 문구: "본 발명은 상기와 같은 종래 기술의 문제점을 해결하기 위해 안출된 것으로..."

#### 05. 과제의 해결수단 (청구항 기초)
- 소스코드의 주요 모듈/클래스/함수 → 발명의 구성요소로 변환
- 데이터 흐름 (입력→처리→출력) 추적
- 방법 발명: "~하는 단계" 형식의 처리 단계 나열
- 시스템 발명: "~을 수행하도록 구성된 {모듈}" 형식

#### 06. 발명의 효과
- 배경기술 문제점 각각에 대응하는 효과 기술
- 가능하면 정량적 근거 포함 (처리 속도 N% 향상, 정확도 N% 개선 등)

#### 07. 발명의 상세한 설명 (최대 분량)
- **시스템 구성** (도면 1 참조) → 각 모듈 상세 설명
- **처리 흐름** (도면 2 참조) → 각 단계 상세 설명
- **데이터 구조** (도면 3 참조) → 필드별 설명
- **실시예 2개 이상** (대표 시나리오 + 변형 시나리오)
- **변형 실시예** (다른 기술 스택, 도메인, 스케일 적용 가능성)
- **핵심 알고리즘** — 의사코드(pseudocode)로 변환하여 기술

#### 08. 청구범위 초안

```
구성:
- 독립항 1: 방법 청구항 
  "~방법에 있어서, (a)~단계; (b)~단계;를 포함하는..."
  
- 종속항 2~4: 독립항 1의 세부 특징
  
- 독립항 5: 시스템 청구항 
  "~시스템에 있어서, ~모듈;을 포함하는..."
  
- 종속항 6~7: 독립항 5의 세부 특징
  
- 독립항 8: 기록매체 청구항 
  "~방법을 실행시키기 위한 프로그램이 기록된..."
```

- **독립항**: 필수 구성요소만 (넓은 권리 범위)
- **종속항**: 세부 특징 추가 (방어력 강화)
- **3종 세트**: 방법 + 시스템 + 기록매체

#### 09. 요약서
- 400자 이내
- 핵심 기술 구성 1~2문장 + 핵심 효과 1문장
- 대표도 번호 지정 (예: "도 2 참조")

#### 10. 도면

**도면 1: 시스템 구성도**
- 모듈 박스 + 데이터 흐름 화살표 + 참조부호(100, 110, 120...)
- SVG 형식

**도면 2: 처리 흐름도**
- 시작(타원) → 처리(직사각형, S210~) → 분기(마름모) → 종료
- 단계별 참조번호 (S100, S110...)

**도면 3: 데이터 흐름/구조도**
- 데이터 입력→변환→저장→출력
- 필드명, 데이터 타입 표시

**KIPO 규칙**:
- 흑백 선화 (컬러 불가)
- 참조부호 아라비아 숫자
- 도면 내 텍스트 최소화

#### 11. 선행기술 조사
- WebSearch로 Google Patents 검색
- KIPRIS 검색 키워드 생성 (한글 + 영문)
- 유사 선행기술별 차별점 분석 표

#### 12. 체크리스트
- 명세서 필수 항목 점검
- 청구범위 논리 검증
- 도면 완성도 확인
- 신규성/진보성 자체 평가
- 출원 절차 안내
- 비용 안내: 출원료 ~46,000원, 심사청구료 ~143,000원, 개인 70% 감면 가능

## 🚀 Phase 3: 실행 흐름

### 자동 실행 순서

```
1. 프로젝트 루트 스캔 → 모듈/함수/클래스 구조 분석
2. 핵심 알고리즘 및 데이터 흐름 추출
3. Git 이력 분석 → 발명 시점 확인
4. README/CLAUDE.md → 프로젝트 목적 및 기술 특징 추출
5. 사용자 정보 입력 (발명자명, 기술분야 등)
6. 선행기술 검색 (Google Patents)
7. patent/ 디렉토리 생성
8. 12개 파일 + 도면 3장 순차 생성
9. 청구범위 자동 초안 작성
10. 결과 보고 및 다음 단계 안내
```

### 실행 예시

```
/patent-spec

→ 프로젝트 스캔 중...

✅ 특허 명세서 자료 생성 완료

발명의 명칭: 지식 그래프 기반 검색 증강 생성을 이용한 소셜 미디어 의견 시뮬레이션 시스템 및 방법
발명 유형: 복합 발명 (방법 + 시스템)
```

## ⚙️ 옵션

| 옵션 | 설명 | 예시 |
|------|------|------|
| `--type=method` | 방법 발명만 (방법 청구항 중심) | `/patent-spec --type=method` |
| `--type=system` | 시스템 발명만 (시스템 청구항 중심) | `/patent-spec --type=system` |
| `--type=all` | 복합 발명 (기본값) | `/patent-spec --type=all` |
| `--prior-art` | 선행기술 조사만 수행 | `/patent-spec --prior-art` |
| `--claims-only` | 청구범위 초안만 생성 | `/patent-spec --claims-only` |
| `--output=path` | 생성 폴더 지정 | `/patent-spec --output=../my-patent` |

## 📌 주의사항

1. **초안 문서** — 이 스킬은 명세서 **초안**을 생성합니다. 최종 출원 전 변리사 검토는 필수입니다.

2. **선행기술 조사** — Google Patents 검색은 기초 조사용입니다. 정식 출원 전 KIPRIS(한국특허정보원) 전체 데이터베이스 조사를 권장합니다.

3. **청구범위 검증** — 생성된 청구범위는 초안입니다. 신규성, 진보성, 명확성 검증이 필요합니다.

4. **기술 분야 선택** — 정확한 IPC 분류를 위해 기술 분야를 정확하게 입력해주세요.

5. **출원 절차** — 한국특허청 특허로(https://www.patent.go.kr)를 통한 전자출원 필수입니다.

### 시나리오 1: 신기술 스타트업
```
→ 핵심 알고리즘 특허 출원 전 명세서 초안 생성
→ 변리사와 함께 내용 검토 및 보완
→ 특허청 출원
```

### 시나리오 2: 오픈소스 프로젝트 기술 보호
```
→ 고유 기술요소 명세서 생성
→ 공개 전 특허 출원 (공개 예약)
→ 오픈소스 공개 후 특허권 행사 가능
```

### 시나리오 3: 기업 핵심 기술 자산화
```
→ 내부 개발 기술 명세서 패키지 생성
→ 특허 포트폴리오 구축
→ 기술가치 평가 자료로 활용 (투자 유치, M&A 등)
```
