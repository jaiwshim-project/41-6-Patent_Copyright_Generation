🚀 시작하기

AI-IP Strategy OS는 Gemini AI를 활용하여 특허 출원과 저작권 등록 과정을 자동으로 고도화하는 플랫폼입니다. 아이디어를 입력하면 AI가 단계적으로 분석하고, 최종 법적 서류를 자동 생성합니다.

기본 사용 흐름

1홈에서 입력제목·아이디어·파일
2유형 선택특허 또는 저작권
3AI 단계 분석각 스텝 실행
4점수 확인PSM·RSI 평가
5서류 완성PDF 출력
💡 Tip: 아이디어 설명은 최소 200자 이상 상세하게 작성할수록 AI 분석 품질이 향상됩니다. 기존 기술과의 차별점, 핵심 작동 원리를 명확히 포함하세요.

🔑 Gemini API Key 설정

플랫폼의 AI 기능을 사용하려면 Google AI Studio에서 발급한 Gemini API Key가 필요합니다.

발급 방법

  • 1단계: Google AI Studio (aistudio.google.com) 접속
  • 2단계: Google 계정으로 로그인
  • 3단계: 상단 메뉴 "Get API Key" 클릭
  • 4단계: "Create API Key" 버튼 클릭
  • 5단계: 발급된 AIzaSy... 형태의 키 복사

입력 방법

  • 상단 내비게이션의 🔑 API Key 설정 버튼 클릭
  • 발급받은 API Key 붙여넣기 후 저장
  • 키는 브라우저 로컬스토리지에만 저장되며 외부로 전송되지 않습니다
⚠ API Key는 개인 정보입니다. 공유 컴퓨터에서는 사용 후 반드시 로컬스토리지를 삭제하세요.

⚙️ 특허 출원 고도화 흐름

특허 출원 고도화는 7단계로 진행되며, 각 단계를 순서대로 실행하면 최종 특허 명세서가 자동으로 완성됩니다.

S1핵심 추출
S2선행기술
S3구성분해
S4청구항
S5실시예
S6점수평가
S7명세서

각 단계의 "실행" 버튼을 클릭하면 Gemini AI가 해당 단계를 분석합니다. 실행 후 "다음" 버튼이 활성화되어 다음 단계로 진행할 수 있습니다.

⚙️ 특허 7단계 상세 설명

STEP 1 · 기술 핵심 추출

AI가 입력된 아이디어에서 발명의 핵심 개념, 기술적 과제, 해결 수단을 자동 추출합니다. 추가 질문(차별점, 상업적 핵심)을 입력하면 분석 품질이 향상됩니다.

STEP 2 · 선행기술 분석

발명과 유사한 기존 기술을 분석하고 차별성을 구조화합니다. 신규성 위험도를 자동 평가하여 보완 방향을 제시합니다.

STEP 3 · 구성요소 분해

발명을 구성하는 핵심 요소(A, B, C...)로 분해하고 작동 흐름을 체계화합니다. 필수 청구항 구성요소와 선택적 요소를 구분합니다.

STEP 4 · 청구항 설계

독립항(방법/시스템/장치), 종속항(15개 이상), 회피설계 방어항을 자동 작성합니다. 한국 특허법 제42조 형식을 준수합니다.

STEP 5 · 실시예 생성

구체적 구현 사례, 변형 실시예, 산업별 적용 사례를 생성하여 권리 범위를 확장합니다.

STEP 6 · 특허 점수 평가

PSM(특허 점수화 모델)과 RSI(권리 강도 지수)를 슬라이더로 조절하여 산출합니다. 거절 확률도 예측합니다.

STEP 7 · 최종 명세서

특허청 제출용 완성 명세서(발명의 명칭~청구범위)를 AI가 자동 작성합니다. PDF 출력이 가능합니다.

📐 PSM 특허 점수화 모델

PSM(Patent Scoring Model)은 특허 출원 전 특허의 강도를 정량 평가하는 100점 기반 모델입니다.

PSM = 0.25×신규성 + 0.20×진보성 + 0.10×산업성
+ 0.10×기술효과 + 0.15×청구항 완성도
+ 0.10×방어력 + 0.10×확장성

각 항목은 1~5점으로 평가하며, 슬라이더를 통해 직접 조정할 수 있습니다.

  • 80점 이상 — 우수: 등록 가능성 높음
  • 60~79점 — 양호: 일부 보완 권장
  • 40~59점 — 보통: 주요 항목 강화 필요
  • 40점 미만 — 미흡: 전면 재설계 검토

💎 RSI 권리 강도 지수

RSI(Right Strength Index)는 특허 등록 이후의 전략적 권리 가치를 평가하는 복합 지수입니다.

RSI = (L × T × M) ÷ V

L = 법적 안정성 (선행기술 충돌 위험 역수)
T = 기술 복잡도 (기술 재현 난이도)
M = 시장 독점성 (시장 진입 장벽)
V = 우회 가능성 (경쟁사의 회피 설계 가능성)
  • 150 이상 — 전략적 핵심 특허
  • 100~150 — 강한 권리
  • 50~100 — 보통 수준
  • 50 미만 — 약한 권리, 강화 필요
💡 RSI를 높이려면 V(우회 가능성)를 낮추는 것이 핵심입니다. 회피설계 방어항을 충분히 추가하고, 다양한 종속항으로 권리 범위를 입체적으로 구성하세요.

©️ 저작권 등록 고도화 흐름

저작권 등록 고도화는 6단계로 진행됩니다.

S1유형 분류
S2창작성 분석
S3창작 경위
S4보호 범위
S5등록 신청서
S6최종 서류

©️ 저작권 6단계 상세 설명

STEP 1 · 창작물 유형 분류

문서, 프로그램 코드, 디자인, 음악, 영상, 플랫폼/시스템 중 해당 유형을 선택합니다. AI가 법적 저작물 유형을 정밀 분류하고 등록 전략을 제시합니다.

STEP 2 · 창작성 분석

표현의 독창성, 구성의 독창성, 아이디어와 표현의 분리 원칙을 분석합니다. 창작성 진술서 초안을 자동 생성합니다.

STEP 3 · 창작 과정 기록

최초 기획일, 제작 과정, 공동 저작자 여부를 입력하면 공식 창작 경위서가 자동 작성됩니다.

STEP 4 · 보호 범위 확정

2차 저작물 허용 여부, 이용 허락 범위 등을 설정합니다. 보호 대상과 제외 요소가 명확히 정의됩니다.

STEP 5 · 등록 신청서 작성

한국저작권위원회 제출용 공식 등록 신청서를 자동 완성합니다.

STEP 6 · 최종 서류 패키지

등록 신청서, 창작성 진술서, 창작 경위서, 보호 범위 명세서를 통합한 서류 패키지가 생성됩니다.

🗂️ 포트폴리오 관리

분석 완료된 프로젝트를 포트폴리오에 저장하여 한눈에 관리할 수 있습니다.

  • 특허/저작권 분석 완료 후 "포트폴리오에 추가" 버튼 클릭
  • 포트폴리오 페이지에서 PSM, RSI, 거절위험 등 핵심 지표 일괄 확인
  • 유형 필터 및 검색으로 원하는 IP 자산을 빠르게 탐색
  • 내보내기 기능으로 TXT 파일 다운로드 가능

📊 대시보드 활용

대시보드는 전체 IP 포트폴리오의 종합 현황을 시각화합니다.

  • 레이더 차트: 특허 강도 6개 항목의 균형을 확인
  • 게이지: PSM, RSI, 거절위험을 직관적으로 표시
  • AI 전략 분석: 포트폴리오 강점·취약점·개선 방향을 AI가 자동 생성
  • 타임라인: 분석 이력과 향후 일정을 관리

❓ 자주 묻는 질문

Q. AI가 생성한 명세서를 그대로 제출할 수 있나요?

A. AI 생성 결과물은 고품질 초안이지만, 실제 제출 전 반드시 전문 변리사의 검토를 받으시길 권장합니다. 본 플랫폼은 변리사 서비스를 대체하지 않습니다.

Q. 입력한 아이디어는 안전하게 보호되나요?

A. 모든 데이터는 브라우저의 로컬스토리지에만 저장되며, 외부 서버에 따로 전송·저장되지 않습니다. Gemini API 호출 시 입력 내용이 Google 서버로 전송되는 점을 유의해주세요.

Q. PSM 점수와 RSI는 공식 지표인가요?

A. PSM과 RSI는 본 플랫폼의 독자적 분석 모델입니다. 특허청 공식 평가 기준이 아니며, 출원 전략 수립을 위한 참고 지표로 활용하시기 바랍니다.

Q. 파일 업로드는 어떻게 처리되나요?

A. 업로드된 파일명만 기록되며, 현재 버전에서는 파일 내용은 AI 분석에 직접 사용되지 않습니다. 텍스트 입력창에 핵심 내용을 직접 입력해주세요.

⭐ 전문가 팁

특허 청구항 작성 팁

  • 독립항은 핵심 기술만 포함, 가능한 넓게 작성
  • 종속항으로 세부 기능을 단계별로 보호
  • 방법 청구항과 시스템 청구항을 모두 작성해 보호 범위 확대
  • 구체적 수치나 특정 소재는 종속항에 배치

RSI를 높이는 전략

  • 기술을 복잡하게 구현하여 T(기술 복잡도) 향상
  • 여러 구성요소의 조합을 청구항에 포함하여 회피 어렵게 구성
  • 핵심 구성요소에 대한 다양한 변형 실시예로 V(우회 가능성) 감소

저작권 등록 팁

  • 창작 과정을 날짜별로 상세히 기록해 두면 분쟁 시 강력한 증거가 됩니다
  • 스케치, 초안, 수정 버전을 보관하여 창작 과정을 입증하세요
  • 코드/소프트웨어는 버전 관리(git)로 창작 이력을 자동 보존할 수 있습니다
